# 网络缓存管理系统 — 接口规范文档

**版本**：v2.0  
**日期**：2026-07-06  
**协议**：RESP（REdis Serialization Protocol）  
**作者**：第四组  

---

## 1. 概述

`CacheServerClient` 是界面层与通信层之间的**核心接口契约**。所有缓存操作均通过此接口完成，界面层不依赖任何具体实现。

支持两种实现，通过 `config.properties` 的 `client.mock` 配置切换：
- `MockCacheClient`（开发/演示模式，client.mock=true）
- `RespCacheClient`（联调/生产模式，client.mock=false）

---

## 2. 接口定义

**包路径**：`com.cache.client.net.CacheServerClient`

### 2.1 连接管理

#### `void connect(String host, int port)`

| 项目 | 说明 |
|------|------|
| 描述 | 连接到缓存服务器 |
| 参数 | `host` — 服务器主机名/IP；`port` — 服务器端口 |
| 返回 | 无 |
| 异常 | 连接失败时抛出 `RuntimeException`（含错误信息） |
| Mock行为 | 仅设置 `connected = true`，不做实际网络连接 |

#### `void disconnect()`

| 项目 | 说明 |
|------|------|
| 描述 | 断开与缓存服务器的连接 |
| 参数 | 无 |
| 返回 | 无 |
| 异常 | 关闭时忽略 IO 异常 |
| Mock行为 | 设置 `connected = false` |

#### `boolean isConnected()`

| 项目 | 说明 |
|------|------|
| 描述 | 检查当前连接状态 |
| 参数 | 无 |
| 返回 | `true` — 已连接；`false` — 未连接 |
| Mock行为 | 返回 `connected` 字段值 |

### 2.2 Server 命令

#### `String ping()`

| 项目 | 说明 |
|------|------|
| 描述 | 测试服务端连通性 |
| 参数 | 无 |
| 返回 | 成功返回 `"PONG"` |
| RESP 请求 | `*1\r\n$4\r\nPING\r\n` |
| RESP 响应 | `+PONG\r\n` |
| Mock行为 | 固定返回 `"PONG"` |

---

### 2.3 String 操作

#### `Optional<String> get(String key)`

| 项目 | 说明 |
|------|------|
| 描述 | 根据 key 获取缓存值（仅 STRING 类型） |
| 参数 | `key` — 缓存键，不能为 null 或空 |
| 返回 | 存在且未过期返回 `Optional.of(value)`；不存在/已过期/类型不匹配返回 `Optional.empty()` |
| 注意 | 如果 key 已过期，实现类应主动删除并返回空 |
| RESP 请求 | `*2\r\n$3\r\nGET\r\n$3\r\nkey\r\n` |
| RESP 响应（存在） | `$5\r\nhello\r\n` |
| RESP 响应（不存在） | `$-1\r\n` |
| Mock行为 | 查询 `stringStore`，检查过期后返回 |

#### `boolean set(String key, String value, long ttlSeconds)`

| 项目 | 说明 |
|------|------|
| 描述 | 设置一个缓存条目（key 已存在则覆盖），可选过期时间 |
| 参数 | `key` — 缓存键；`value` — 缓存值；`ttlSeconds` — 过期秒数，≤0 永不过期 |
| 返回 | `true` — 成功 |
| RESP 请求（无 TTL） | `*3\r\n$3\r\nSET\r\n$3\r\nkey\r\n$5\r\nvalue\r\n` |
| RESP 请求（带 TTL） | `*5\r\n$3\r\nSET\r\n$3\r\nkey\r\n$5\r\nvalue\r\n$2\r\nEX\r\n$2\r\n60\r\n` |
| RESP 响应 | `+OK\r\n` |
| Mock行为 | 写入 `stringStore`，如果同名 key 在 `listStore` 中存在则先移除 |

### 2.4 Key 操作

#### `int del(String... keys)`

| 项目 | 说明 |
|------|------|
| 描述 | 删除一个或多个键（支持变参） |
| 参数 | `keys` — 变参，一个或多个键名 |
| 返回 | 实际删除的键数量（不存在的 key 不计数） |
| RESP 请求 | `*3\r\n$3\r\nDEL\r\n$2\r\nk1\r\n$2\r\nk2\r\n` |
| RESP 响应 | `:2\r\n` |
| Mock行为 | 同时从 `stringStore` 和 `listStore` 中移除，返回移除总数 |
| 注意 | 旧版使用 `delete(String)` 返回 boolean；新版改为 `del(String...)` 返回 int |

#### `long ttl(String key)`

| 项目 | 说明 |
|------|------|
| 描述 | 查询 key 的剩余生存时间 |
| 参数 | `key` — 要查询的键 |
| 返回 | `n` — 剩余秒数（≥0）；`-1` — key 存在但永不过期；`-2` — key 不存在 |
| RESP 请求 | `*2\r\n$3\r\nTTL\r\n$3\r\nkey\r\n` |
| RESP 响应 | `:60\r\n` 或 `:-1\r\n` 或 `:-2\r\n` |
| Mock行为 | 从 `stringStore` 获取 entry 计算 `createTime + ttlSeconds - now` |
| 注意 | TTL 返回码约定（-1/-2）已由第二组确认，与标准 Redis 语义一致 |

### 2.5 List 操作

#### `int lpush(String key, String... values)`

| 项目 | 说明 |
|------|------|
| 描述 | 从列表左侧（头部）插入一个或多个元素 |
| 参数 | `key` — 列表名；`values` — 要插入的值 |
| 返回 | 插入后的列表总长度 |
| RESP 请求 | `*4\r\n$5\r\nLPUSH\r\n$1\r\nq\r\n$1\r\na\r\n$1\r\nb\r\n$1\r\nc\r\n` |
| RESP 响应 | `:3\r\n` |
| Mock行为 | 在 `listStore` 中对应列表头部插入，自动创建不存在的列表 |
| 注意 | 后插入的元素在前面（堆栈语义）：`lpush q a b c` → 列表为 `[c, b, a]` |

#### `int rpush(String key, String... values)`

| 项目 | 说明 |
|------|------|
| 描述 | 从列表右侧（尾部）插入一个或多个元素 |
| 参数 | `key` — 列表名；`values` — 要插入的值 |
| 返回 | 插入后的列表总长度 |
| RESP 请求 | `*4\r\n$5\r\nRPUSH\r\n$1\r\nq\r\n$1\r\na\r\n$1\r\nb\r\n$1\r\nc\r\n` |
| RESP 响应 | `:3\r\n` |
| Mock行为 | 在 `listStore` 中对应列表尾部追加，自动创建不存在的列表 |
| 注意 | 保持插入顺序（队列语义）：`rpush q a b c` → 列表为 `[a, b, c]` |

#### `String lpop(String key)`

| 项目 | 说明 |
|------|------|
| 描述 | 从列表左侧（头部）弹出一个元素 |
| 参数 | `key` — 列表名 |
| 返回 | 弹出的值；列表为空或不存在返回 `null` |
| RESP 请求 | `*2\r\n$4\r\nLPOP\r\n$1\r\nq\r\n` |
| RESP 响应（有值） | `$1\r\na\r\n` |
| RESP 响应（空） | `$-1\r\n` |
| Mock行为 | 从 `listStore` 头部移除并返回；弹出最后一个元素时自动删除列表 |

#### `List<String> lrange(String key, int start, int stop)`

| 项目 | 说明 |
|------|------|
| 描述 | 获取列表指定范围的元素（闭区间） |
| 参数 | `key` — 列表名；`start` — 起始索引（0-based）；`stop` — 结束索引（支持负索引，-1 表示最后） |
| 返回 | 元素列表；列表不存在返回空列表 |
| RESP 请求 | `*4\r\n$6\r\nLRANGE\r\n$1\r\nq\r\n$1\r\n0\r\n$2\r\n-1\r\n` |
| RESP 响应 | `*3\r\n$1\r\na\r\n$1\r\nb\r\n$1\r\nc\r\n` |
| Mock行为 | 从 `listStore` 中获取子列表，处理负索引和越界 |

### 2.6 SCAN 操作

#### `List<String> scan(String pattern)`

| 项目 | 说明 |
|------|------|
| 描述 | 遍历匹配模式的键列表（格式已由第二组确认） |
| 参数 | `pattern` — 通配符模式（支持 `*`），null 或 `"*"` 返回所有 key |
| 返回 | 匹配的 key 列表；无匹配返回空列表 |
| RESP 请求 | `*1\r\n$4\r\nSCAN\r\n`（全部）或 `*2\r\n$4\r\nSCAN\r\n$5\r\nuser*\r\n` |
| RESP 响应 | `*N\r\n$len1\r\nkey1\r\n$len2\r\nkey2\r\n...` 或 `*0\r\n`（无匹配） |
| Mock行为 | 从 `allKeys` 集合中过滤匹配 pattern 的 key |
| 注意 | 非游标式，一次性返回全部匹配 key |

### 2.7 重连操作

#### `boolean reconnect()`

| 项目 | 说明 |
|------|------|
| 描述 | 重新连接到上次连接的服务器（先关闭旧连接，再创建新连接） |
| 参数 | 无 |
| 返回 | `true` — 重连成功；`false` — 重连失败（从未连接过或连接失败） |
| Mock行为 | 设置 `connected = true`，返回 `true` |

#### `String getLastHost()`

| 项目 | 说明 |
|------|------|
| 描述 | 获取上次成功连接的服务器地址 |
| 参数 | 无 |
| 返回 | 主机名/IP 字符串；Mock 模式返回 `"mock"` |

#### `int getLastPort()`

| 项目 | 说明 |
|------|------|
| 描述 | 获取上次成功连接的服务器端口 |
| 参数 | 无 |
| 返回 | 端口号；Mock 模式返回 `0` |

## 3. 数据模型

### `CacheEntry`

**包路径**：`com.cache.client.model.CacheEntry`

#### 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | String | 缓存键 |
| `type` | EntryType | 数据类型枚举（`STRING` / `LIST`） |
| `value` | String | STRING 类型的缓存值 |
| `ttlSeconds` | long | 过期时间（秒），≤0 表示永不过期 |
| `createTime` | Instant | 条目创建时间 |
| `listLength` | int | LIST 类型的元素个数（仅 LIST 有效） |

#### `EntryType` 枚举

| 值 | 说明 |
|:---:|------|
| `STRING` | 字符串类型（对应 SET/GET 命令） |
| `LIST` | 列表类型（对应 LPUSH/RPUSH/LPOP/LRANGE 命令） |

#### 方法

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `isExpired()` | boolean | `ttlSeconds > 0` 且当前时间已超过 `createTime + ttlSeconds` 返回 true |
| `getRemainingTtl()` | long | 剩余秒数；永不过期返回 -1；已过期返回 0 |

## 4. 实现类规范

### 4.1 MockCacheClient

| 项目 | 说明 |
|------|------|
| 用途 | 开发/演示阶段使用，不依赖服务端 |
| 数据存储 | `ConcurrentHashMap<String, CacheEntry>` stringStore + `ConcurrentHashMap<String, List<String>>` listStore + `Set<String>` allKeys |
| 预置数据 | 5 条 String + 2 条 List 示例数据 |
| 线程安全 | 是（ConcurrentHashMap + CopyOnWriteArrayList） |
| 过期处理 | `get()` / `getAllLocalEntries()` 时检查过期并清除 |
| 特有能力 | `getAllLocalEntries()` 返回全部条目的本地快照（不在接口上，供 UI 直接渲染） |

### 4.2 RespCacheClient

| 项目 | 说明 |
|------|------|
| 用途 | 联调/生产阶段使用，通过 RESP 协议与 Linux 服务端通信 |
| 连接方式 | `java.net.Socket`，由 `connect(host, port)` 建立 |
| 通信协议 | **RESP 文本协议**（REdis Serialization Protocol） |
| 编解码 | 委托 `com.cache.client.net.protocol.RespCodec` |
| 响应封装 | `com.cache.client.net.protocol.RespResponse` |
| 粘包处理 | 通过流式解析，按头字节（+ - : $ \*）分支判断 |
| 待实现 | `scan()` 和 `ttl()` 待第二组格式确认后补全 |

## 5. 使用示例

```java
// 通过全局工厂获取客户端实例
CacheServerClient client = CacheClientApp.getClient();

// 连接
client.connect("127.0.0.1", 6379);

// PING 测试
String pong = client.ping();  // "PONG"

// String 操作
client.set("username", "Alice", 300);
Optional<String> value = client.get("username");  // "Alice"

// Key 操作（变参）
int deleted = client.del("username", "tempkey");  // 返回实际删除数量

// List 操作
client.lpush("queue", "a", "b", "c");
int len = client.rpush("queue", "d");               // 4
String head = client.lpop("queue");                 // "c"
List<String> all = client.lrange("queue", 0, -1);   // ["b", "a", "d"]

// TTL 查询
long ttl = client.ttl("username"); // 剩余秒数 / -1(持久) / -2(不存在)

// 断开
client.disconnect();
```

## 6. 响应类型与 RESP 格式速查

| 客户端方法 | 服务端命令 | RESP 请求示例 | RESP 响应类型 |
|-----------|-----------|--------------|:------------:|
| `ping()` | PING | `*1\r\n$4\r\nPING\r\n` | `+PONG\r\n` |
| `set(k, v)` | SET | `*3\r\n$3\r\nSET\r\n...` | `+OK\r\n` |
| `set(k, v, ex)` | SET EX | `*5\r\n$3\r\nSET\r\n...\r\n$2\r\nEX\r\n$2\r\n60\r\n` | `+OK\r\n` |
| `get(k)`（存在） | GET | `*2\r\n$3\r\nGET\r\n$3\r\nkey\r\n` | `$5\r\nvalue\r\n` |
| `get(k)`（不存在） | GET | 同上 | `$-1\r\n` |
| `del(k1, k2)` | DEL | `*3\r\n$3\r\nDEL\r\n...` | `:2\r\n` |
| `ttl(k)` | TTL | `*2\r\n$3\r\nTTL\r\n...` | `:60\r\n` / `:-1\r\n` / `:-2\r\n` |
| `lpush(k, v1, v2)` | LPUSH | `*4\r\n$5\r\nLPUSH\r\n...` | `:3\r\n` |
| `rpush(k, v1, v2)` | RPUSH | `*4\r\n$5\r\nRPUSH\r\n...` | `:3\r\n` |
| `lpop(k)`（有值） | LPOP | `*2\r\n$4\r\nLPOP\r\n...` | `$1\r\na\r\n` |
| `lpop(k)`（空） | LPOP | 同上 | `$-1\r\n` |
| `lrange(k, 0, -1)` | LRANGE | `*4\r\n$6\r\nLRANGE\r\n...` | `*3\r\n$1\r\na\r\n...` |
| `reconnect()` | 重连（客户端侧） | 无 RESP 命令，客户端内部实现 | `true` / `false` |
```
