## 通用机制说明

### 异步执行框架

所有网络操作统一通过 `MainController.runAsync()` 方法执行，避免阻塞 JavaFX UI 线程：

```
参数传递:
  callable    → 后台线程执行的 Callable<T>（网络 IO 在此发生）
  onSuccess   → 后台执行成功后的 UI 更新回调 Consumer<T>
  errorTitle  → 失败时弹窗标题（null 则不弹窗）

流程:
  runAsync() 内部创建 javafx.concurrent.Task<T>
    → task.call() 在后台线程执行 callable.call()
    → task.setOnSuccess() 在 JavaFX 线程执行 onSuccess.accept(result)
    → task.setOnFailed() 在 JavaFX 线程弹错误提示
```

### 双客户端模式

所有通过网络发起的操作都经过 `CacheServerClient` 接口，根据 `config.properties` 中 `client.mock=true/false` 切换：

| 模式 | 实现类 | 行为 |
|:----:|--------|------|
| Mock | `MockCacheClient` | 本地内存操作，不依赖网络 |
| RESP | `RespCacheClient` | 通过 TCP Socket 发送 RESP 协议命令到服务端 |

## 1. 连接服务器

**触发**：用户在 `serverHostField` / `serverPortField` 输入地址和端口 → 点击"Connect"按钮

```
FXML 绑定:
  MainView.fxml 中 Button.onAction → MainController#onConnect
```

### step-by-step 调用链

```
[1] MainView.fxml (Button)
    onAction 事件
    ↓
[2] MainController.onConnect()                          MainController.java:273
    ├─ 读取 serverHostField.getText()                    ← 用户输入的主机名
    │   └─ 若为空 → 使用 Config.getDefaultHost()         ← config.properties → "127.0.0.1"
    ├─ 解析 serverPortField.getText() 为 int             ← 用户输入的端口
    │   └─ 若无效 → 使用 Config.getDefaultPort()         ← config.properties → 6379
    ├─ 变量: connectHost (String), connectPort (int)
    │
    └─ runAsync(
           callable:  () -> { client.connect(connectHost, connectPort); return true; },
           onSuccess: ok -> {
               connectedHost = connectHost;              ← 保存连接信息
               connectedPort = connectPort;
               connectionStatusLabel.setText("Connected...");
               startHeartbeat();                         ← 启动 5 秒心跳
           },
           errorTitle: "Connection failed"
       )
```

### RESP 模式下 connect 内部

```
[3] RespCacheClient.connect(host, port)                  RespCacheClient.java:46
    ├─ 加锁 synchronized(lock)
    ├─ int timeout = Config.getConnectTimeout()           ← config.properties → 5000ms
    ├─ socket = new Socket()
    ├─ socket.connect(new InetSocketAddress(host, port), timeout)
    ├─ socket.setSoTimeout(timeout)
    ├─ in  = socket.getInputStream()
    ├─ out = socket.getOutputStream()
    ├─ connected = true
    └─ lastHost = host;  lastPort = port                 ← 供重连使用
```

### Mock 模式下 connect 内部

```
[3] MockCacheClient.connect(host, port)                  MockCacheClient.java:82
    └─ connected = true
```

[1] MainController.onRemoveValue()                        MainController.java:501
    ├─ String key   = listKeyField.getText().trim()

    ├─ if (key.isEmpty() || value.isEmpty()) return
    │
    └─ runAsync(
           callable: () -> {
               // 客户端侧实现：取出 → 过滤 → 重建
               List<String> all = client.lrange(key, 0, -1);
               List<String> remaining = all.stream()
                   .filter(v -> !v.equals(value))           ← 过滤掉匹配值

                   if (!remaining.isEmpty())
                       client.rpush(key, remaining.toArray(new String[0])); ← 重建
               }
               return removed;
           },
           onSuccess: removed -> {
               if (removed == 0) showError(...);           ← 值不存在
               else { refreshListDisplayAsync(key); refreshTableAsync(); }
           },
